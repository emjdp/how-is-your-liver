# CALCULATION_RULES — 계산 규칙

> 본 계산은 의료 진단이 아니라 엔터테인먼트·참고용 추정치다. 모든 결과 페이지·카드에 안전 안내 문구를 고정 표시한다.

## 계산 상수 (`lib/constants.ts`)

```ts
export const DRINKS = {
  soju: { mlPerUnit: 360, abv: 0.165, unitLabel: "병" },
  beer: { mlPerUnit: 500, abv: 0.045, unitLabel: "잔" },
} as const;

export const ALCOHOL_DENSITY_G_PER_ML = 0.789;

// 시간당 알코올 대사량 (보수적 평균치 — 개인차 큼)
// 일반 성인 약 7~10g/h. 처리 추정 시간 표시는 8g/h 기준 계산 + ±25% 범위를 hint로 제공.
export const METABOLISM_G_PER_HOUR = 8;

export const KCAL_PER_G_ALCOHOL = 7;

// 입력 제한
export const INPUT_MAX = { soju: 30, beer: 30 } as const;
```

## 순수 알코올 g
```
pureAlcoholG(volumeMl, abv) = volumeMl * abv * 0.789
```
유도된 상수:
- 소주 1병 = 360 × 0.165 × 0.789 ≈ **46.87 g**
- 맥주 1잔 = 500 × 0.045 × 0.789 ≈ **17.75 g**

## 소주 환산 병 수
```
sojuEquivBottles(totalAlcoholG) = totalAlcoholG / 46.87
```
표시 자리수: 소수점 1자리.

## 칼로리 (kcal)
```
kcal(totalAlcoholG) = totalAlcoholG * 7
```
표시 자리수: 정수 반올림.

## 알코올 처리 추정 시간
> ⚠️ 명칭 주의: "회복 시간"이 아니라 **"알코올 처리 추정 시간"** (UI 라벨: `처리 추정 시간`).
> 사용자가 "술이 깬 시간 = 운전 가능 시점"으로 오해하지 않도록 명칭을 분리한다.

```
hours = totalAlcoholG / 8
displayHours = round(hours * 2) / 2   // 0.5h 단위 반올림
```
- 표시 예: "약 5.5시간"
- hint 라인 (작게): "개인차에 따라 ±25%"
- **모든 표시 옆에 고정 부속 카피**: "운전 가능 시점이 아닙니다."

## 반올림 규칙
- 소주 환산 병: 소수점 1자리 (`Math.round(x*10)/10`).
- 칼로리: 정수.
- 처리 추정 시간: 0.5h 단위.
- 순수 알코올 g: 소수점 1자리.

## 입력 정책
- **정수만 허용.** 소주 0.5병 같은 소수 입력은 MVP에서 미지원.
  - 이유: 입력 속도(±버튼 한 손 조작) 우선, 추정치에 0.5병 단위는 의미 없음.
  - 추후 확장 시 "반병" 토글 등을 별도 검토.

## 0 입력 vs 기록 없음 — 정책 통일
"기록이 없는 날"과 "0으로 저장된 기록"을 데이터·UI에서 명확히 구분한다.

| 상황 | 데이터 상태 | 결과 페이지 | 주간 합산 | 비고 |
|---|---|---|---|---|
| 기록 없음 | `records[date] === undefined` | **메인 리다이렉트** + 토스트 "이 날의 기록이 없어요." | 합산에서 제외(분모 영향 X) | 사용자가 한 번도 손대지 않은 날 |
| 0으로 저장 | `records[date] = {soju:0, beer:0, ...}` | **정상 표시** — `t0_peace`(간 평화주의자), 카피 "오늘 간은 휴가 중입니다." | 합산에는 0 포함하되 음주일 수에는 미포함 | 사용자가 일부러 0으로 기록한 날 ("쉬었음") |

- 입력 카드에서 ± 버튼을 한 번이라도 눌러 값을 변경하면(0→1→0 포함) `DayRecord`가 생성·저장된다.
- 단순 진입(±터치 없음) 후 빠져나가면 `DayRecord`가 생성되지 않는다.
- 음주일 수 계산식: `records.filter(r => r.soju > 0 || r.beer > 0).length`.

## 비정상 입력 방어
- **음수**: UI에서 차단 (− 버튼 disabled at 0). 데이터 레이어에서도 `Math.max(0, n)`.
- **과도값**: `Math.min(n, INPUT_MAX.soju)` / `INPUT_MAX.beer`. UI에서 + 버튼 disabled at max.
- **NaN/undefined**: 0으로 폴백.
- **LocalStorage 파손 데이터**: try/catch 후 해당 날짜를 0으로 폴백 + `console.warn`.

## 예시 계산
- 소주 2병 + 맥주 3잔
  - 알코올 = 2 × 46.87 + 3 × 17.75 = 93.74 + 53.25 = **146.99 g**
  - 소주 환산 = 146.99 / 46.87 ≈ **3.1병**
  - kcal = 147 × 7 ≈ **1029 kcal**
  - 처리 추정 시간 = 147 / 8 ≈ 18.4 → **약 18.5시간** (운전 가능 시점 아님)

## UI 표시 방식
- 모든 수치 옆에 단위 표시 (`병`, `g`, `kcal`, `시간`).
- 페이지마다 `*참고용` 캡션을 1회 노출.
- **처리 추정 시간 옆에 항상 인접 표시**:
  - "운전 가능 시점이 아닙니다." (작게, `--color-muted`).
  - 작은 정보 아이콘 → 탭하면 모달: "알코올 처리 추정 시간은 보수적 평균치(8g/h)이며, 개인의 체중·성별·음식 섭취·건강 상태에 따라 크게 달라집니다. 운전 가능 여부 판단에는 사용할 수 없습니다."

## 참고용/엔터테인먼트 안내
- 모든 계산 결과 페이지 하단에 [TONE_GUIDE](./TONE_GUIDE.md) 의 안전 안내 1줄 고정.
- 본 서비스는 의료 서비스가 아니며, 운전 가능 여부 판정에 사용 금지.
