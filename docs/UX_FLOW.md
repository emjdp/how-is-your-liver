# UX_FLOW — 사용자 흐름

## 전체 사용자 흐름 (Happy Path)

```
[첫 진입]
   └─ 메인 화면 즉시 진입 → today 자동 선택
[메인 화면]
   ├─ 요일바: today 선택 상태
   ├─ 입력 카드: 소주 0, 맥주 0
   ├─ 사용자 ± 입력 → 자동 저장 + 미니 티어 갱신
   └─ [이 날의 간 상태 보기] CTA → 당일 리포트
[당일 리포트]
   ├─ 큰 숫자(소주환산 / 알코올 g / 처리 추정 시간)
   ├─ 티어 + 블랙코미디 한 줄
   ├─ [스토리 카드 만들기] / [기록 수정] / [주간 리포트 보기]
[스토리 카드]
   ├─ 2종 템플릿 가로 스와이프 (MVP)
   ├─ [저장] (Share API or Download)
   └─ [돌아가기]
[주간 리포트]
   ├─ 7일 합산 + 막대 그래프
   ├─ 주간 티어
   └─ [주간 스토리 카드 만들기] 비활성 / 추후 제공
```

## 메인 화면 흐름
- 진입 시점: `useEffect` 안에서 KST today를 계산해 요일을 자동 선택한다.
- 요일 탭 시: 입력 카드가 페이드 + 슬라이드로 해당 날짜 기록으로 전환된다 (Framer Motion).
- 입력 변경 시: 500ms 디바운스 후 `localStorage.setItem`. 디바운스 중에는 카드 우상단에 작은 dot indicator + 스크린리더용 `aria-live="polite"` 라벨 ("저장 중" → "저장됨")로 상태를 알린다.
- ± 버튼을 한 번이라도 눌러 값이 변경되면 `DayRecord` 객체가 생성되어 저장된다(0→1→0 포함). 단순 진입 후 버튼 미조작 상태면 객체 자체가 생성되지 않는다.

## 요일 선택 흐름
- **최근 7일 슬라이딩 윈도우** (today 포함 6일 전 ~ today).
  - 이유: 학생들은 주 단위가 아니라 "어제/그저께" 단위로 사고함. 일요일에 들어왔을 때 "이번주 월요일" 데이터가 안 보이면 어색.
- 각 요일 셀: 한글 요일(월/화/...) + 일자(숫자).
- today 셀: 라임 포인트 컬러 + 둥근 인디케이터.
- 선택 셀: 진한 글래스 배경 + 가는 외곽선.
- 기록 있는 셀: 셀 하단 1px 라임 닷.

## 음주량 입력 흐름
- 카드 상단: "5월 17일 토요일" 형식의 날짜 라벨.
- 두 행:
  - **소주** [−][숫자][+] — 큰 숫자 + 작은 ml 환산 hint
  - **맥주** [−][숫자][+]
- 하단 미니 티어: "현재: 야장 정규직 — 공유는 가능합니다."
- CTA 2개: `[이 날의 간 상태 보기]` (primary), `[최근 7일 간 리포트 보기]` (ghost).

## 당일 결과 흐름
- 라우트: `/result?d=YYYY-MM-DD` (검색 파라미터).
- **빈 기록 vs 0 기록**:
  - `records[date]` 객체가 **존재하지 않으면** → 메인 리다이렉트 + 토스트 "이 날의 기록이 없어요."
  - `records[date]` 객체가 **존재하면** (0/0 포함) → 정상 표시. 0/0이면 `t0_peace` 카피로 표시.
- 섹션 순서:
  1. 날짜·요일 헤더 (작게)
  2. 티어 배지 + 메인 카피 (크게, headline)
  3. 핵심 수치 3개 (소주환산 병 · 순수 알코올 g · **처리 추정 시간**). 처리 추정 시간 옆에 "운전 가능 시점 아님" 부속 카피 + 정보 아이콘.
  4. 상세 (소주/맥주 입력 내역 · 칼로리)
  5. 안전 안내 (페이지용 긴 버전 1줄 고정)
  6. CTA: `[스토리 카드 만들기]`, `[기록 수정]`, `[최근 7일 간 리포트 보기]`
- **높은 티어(t5/t6) 시**: `[스토리 카드 만들기]` 라벨은 그대로지만 보조 카피를 차분히("공유는 신중히") 유지. 공유 유도를 강조하지 않는다.

## 주간 리포트 흐름
- 라우트: `/weekly`.
- 모든 본문 표현은 "이번 주" 대신 **"최근 7일"** 로 통일.
- 섹션:
  1. "최근 7일 간 정산"
  2. 주간 티어 + 카피
  3. 핵심 수치 4개 (총 알코올 g · 소주환산 · 음주일 수 · 최다 음주일)
     - **음주일 수** 계산: `soju > 0 || beer > 0` 인 날의 수.
     - **최다 음주일 동률 처리**: 동률 시 **today에 가장 가까운 날**을 우선 표시. 동률이 3일 이상이면 "최다 음주일: 여러 날 동률" 로 표기.
  4. 7일 막대 차트 (CSS only, lib 없음). 기록 없는 날 = 회색 표시, 0/0 저장 = 0 높이 라임 라벨.
  5. 안전 안내 (긴 버전)
  6. CTA: ~~`[주간 스토리 카드 만들기]`~~ MVP에서는 비활성 상태로 자리만 잡거나, 클릭 시 "스토리 카드는 곧 제공됩니다." 토스트. (추후 확장)

## 스토리 카드 생성 흐름
- 라우트: `/result/card?d=...` (당일만). 주간 카드는 추후 확장.
- 화면: 9:16 미리보기 가운데 + 좌우 템플릿 스와이프 인디케이터(MVP는 **2종**).
- 하단: `[저장하기]` (단일 CTA). 누르면:
  - `navigator.canShare?.({files: [pngFile]})` 가능 → 시스템 공유 시트.
  - 미지원 → `<a download>` 클릭 트리거.
- 저장 직후 토스트:
  - 낮은~중간 티어: "저장 완료 — 스토리에 올려보세요."
  - 높은 티어(t5/t6): "저장 완료." (공유 유도 X)

## 빈 상태 / 오류 상태
- **빈 기록 날짜에서 결과 보기 시도** (`records[date]` 없음): 메인으로 리다이렉트 + 토스트 "이 날의 기록이 없어요."
- **0/0 저장 기록**: 결과 페이지 정상 진입, `t0_peace` 카피 ("오늘 간은 휴가 중입니다.")
- **주간 기록 0건**: 주간 리포트 페이지가 "최근 7일은 간이 조용히 쉬었네요." 카피 + 빈 차트.
- **LocalStorage 미지원/시크릿 모드**: 진입 시 상단 배너 — "기록이 저장되지 않을 수 있어요."
- **카드 생성 실패**: 토스트 + 재시도 버튼.

## CTA 구조 (전 페이지 공통)
- **Primary**: 가장 진행 의도가 강한 1개 (예: 결과 보기, 카드 만들기).
- **Secondary**: ghost / text 스타일, 보조 동선.
- 동급 버튼 3개 이상 배치 금지.
