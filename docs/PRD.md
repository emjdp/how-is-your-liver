# PRD — 당신의 간은 안녕하십니까?

## 제품 목표
- 술 마신 직후·다음 날, 자신의 음주량을 빠르고 정직하게 마주보게 하는 모바일 웹 경험을 제공한다.
- 공유 가능한 카드형 출력으로 자연 확산을 만든다.
- 절주 강요 없이, 자기 객관화의 계기를 가볍게 던진다.

## 사용자 문제
1. "어제 얼마나 마셨지?" — 잔/병 단위는 추상적이고, 건강 영향은 더 모름.
2. "스토리에 뭐라도 올리고 싶다" — 단순 사진보다 콘셉트 있는 카드를 원함.
3. "절주 앱은 너무 부담스럽다" — 진지한 헬스 앱은 진입조차 안 함.

## 핵심 기능 (MVP)
| # | 기능 | 요약 |
|---|---|---|
| F1 | 요일 선택 바 | 최근 7일 슬라이딩 윈도우, 오늘 자동 선택, 기록일 닷 표시 |
| F2 | 음주 입력 카드 | 소주 병 수 / 맥주 잔 수, ± 버튼, 자동 저장 |
| F3 | 실시간 미니 티어 | 입력 즉시 간이 티어 + 카피 1줄 |
| F4 | 당일 리포트 | 순수 알코올 g / 소주환산 / kcal / **알코올 처리 추정 시간** / 티어 |
| F5 | 주간 리포트 | 최근 7일 합산, 음주일 수, 최다 음주일, 주간 티어 |
| F6 | 스토리 카드 생성 | **MVP 2종 템플릿** (`tpl_report`, `tpl_overtime`), 당일 결과 전용, 1080×1920 PNG 저장. 주간 카드와 나머지 3종은 추후 확장 |
| F7 | 카드 공유 | Web Share API 우선, 미지원 시 다운로드 |
| F8 | 안전 안내 | 모든 리포트·카드에 고정 표시 (페이지용 ≤80자 / 카드용 ≤32자 분리) |

## MVP 범위 (포함)
- F1~F8 (F6은 2종 템플릿).
- 다크 모드 자동 대응 (`prefers-color-scheme`), 토글 UI 없음.
- 기본 모바일 메타 (favicon, theme-color, mobile viewport). 본격 PWA(매니페스트·서비스워커·오프라인)는 미포함.
- 기록 수정·0 처리. 사용자가 0으로 저장한 날도 결과 페이지에서 정상 표시 (`t0_peace`).

## 제외 범위 (추후 확장)
- 로그인, 서버 DB, Supabase, Firebase.
- 학교별/지역별 랭킹, 친구 비교, 커뮤니티.
- GPS·술집 추천·야장 지도.
- 와인/막걸리/위스키/하이볼 등 추가 주종.
- **카드 템플릿 3종 추가** (`tpl_forecast`, `tpl_weekly`, `tpl_warning`).
- **주간 스토리 카드** 생성.
- 사용자 사진 배경 업로드 카드 (설계만 [STORY_CARD_SPEC](./STORY_CARD_SPEC.md), 구현·stub 모두 X).
- AI 이미지 생성.
- 의학적 진단/BAC 기반 운전 가능 판정.
- JSON export (LocalStorage 백업), 다크 모드 토글, 본격 PWA.
- shadcn/ui 도입.
- 실시간 전국 통계, 결제, 회원가입, 관리자 페이지.

## 기능 요구사항 (FR)
- **FR-1** 첫 진입 시 KST 기준 today 요일이 자동 선택된다.
- **FR-2** 요일바는 최근 7일 슬라이딩 윈도우(today 포함, 6일 전까지). "오늘" 라벨이 시각적으로 강조된다. UI 본문 표현은 "이번 주" 대신 **"최근 7일"** 로 통일한다.
- **FR-3** 입력 카드는 소주 0~30(정수), 맥주 0~30(정수) 범위에서 ± 버튼으로 조작한다. 직접 숫자 입력은 부가 기능. 소수 입력은 MVP 미지원.
- **FR-4** 입력 변경 시 100ms 내 미니 티어가 갱신된다.
- **FR-5** 자동 저장: 입력 후 500ms 디바운스로 LocalStorage에 기록한다. ± 버튼이 한 번이라도 눌리면 `DayRecord` 객체가 생성된다.
- **FR-6** 결과 페이지는 선택된 날짜의 `DayRecord` 객체가 존재할 때만 렌더링한다.
  - 객체 없음 → 메인 리다이렉트 + 토스트.
  - 객체 있음(0/0 포함) → 정상 표시.
- **FR-7** 주간 리포트는 today 포함 최근 7일 (KST 0시 기준)을 사용한다. 음주일 수는 `soju>0 || beer>0`, 최다 음주일 동률은 today에 가까운 날 우선.
- **FR-8** 스토리 카드 생성은 클라이언트 사이드 `html-to-image` 사용, 최소 1080×1920 png. MVP는 당일 결과 페이지에서 **2종 템플릿**만.
- **FR-9** 공유: `navigator.share` 지원시 파일 공유, 미지원시 다운로드 링크. 높은 티어(t5/t6)는 공유 유도 카피 톤을 낮춘다.
- **FR-10** 모든 리포트·카드 하단에 안전 안내 문구를 고정 표시한다. 처리 추정 시간 옆에는 "운전 가능 시점 아님" 부속 카피를 인접 배치한다.

## 비기능 요구사항 (NFR)
- **성능**: First Contentful Paint < 1.5s (4G), 카드 생성 < 1.5s.
- **접근성**: 모든 ± 버튼 최소 44×44px, 색 대비 WCAG AA 이상.
- **호환성**: iOS Safari 16+, Chrome Android 최신 2버전.
- **국제화**: 1차 한국어 only. 카피는 모두 `data/` 디렉토리에서 관리(추후 i18n 전환 용이).
- **개인정보**: 서버 전송 0건. LocalStorage만 사용.

## 성공 지표
- **정성**: 사용자 인터뷰(8명)에서 "올리고 싶다" / "현타 온다" 양쪽 모두 60% 이상 언급.
- **정량**(분석 미연동 MVP — 자체 추정): 카드 저장 후 30분 내 인스타 스토리 노출 사례를 친구 그룹 모니터링으로 1건 이상 확인.

## 안전/윤리 고려사항
- 본 서비스는 성인 음주를 전제로 한다. 진입 게이트(연령 확인 모달)는 두지 않지만, 모든 카피·이미지·아이콘은 성인 문맥에서만 사용한다.
- 술 운전 관련 농담·기능 전면 금지.
- 의학 진단처럼 보이는 표현 금지 (구체 사례는 [TONE_GUIDE](./TONE_GUIDE.md) 참조).
- 임신·복약 등 위험군 언급은 MVP에서 다루지 않고, 안전 안내에서 "일반 성인 기준 참고용"이라고만 한정한다.
